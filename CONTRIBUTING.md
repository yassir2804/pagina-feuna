# Guía de Contribución (Interna FEUNA/DTIC)

Este repositorio es de uso interno para el equipo FEUNA/DTIC.
No está pensado como proyecto open source público.

## Alcance

- Equipo esperado: 2 a 5 personas.
- Objetivo: mantener y evolucionar el sitio oficial FEUNA.
- Canal de coordinación: chat interno y reuniones de seguimiento.

## Flujo simple de trabajo

1. Crear rama desde main.
2. Hacer cambios pequeños y claros.
3. Ejecutar pruebas mínimas:
   - npm run build
   - revisión visual rápida en móvil y desktop
4. Subir cambios con commit descriptivo.
5. Abrir PR cuando el cambio sea grande; para cambios pequeños, se puede trabajar directo en main si el equipo lo acuerda.

## Convención de ramas

- feature/nombre-corto
- fix/nombre-corto
- docs/nombre-corto

## Convención de commits

Formato recomendado:

- feat: nuevo componente o funcionalidad
- fix: corrección de error
- docs: cambios de documentación
- chore: mantenimiento o limpieza

Ejemplo:

- fix: corregir enlaces internos de navegacion

## Reglas mínimas de calidad

- Mantener consistencia visual con FEUNA.
- Evitar rutas rotas y enlaces que den 404.
- Mantener textos claros y lenguaje institucional.
- No subir credenciales ni tokens al repositorio.

## Contacto interno

Para dudas de contenido, diseño o publicación final, coordinar con FEUNA y DTIC por los canales institucionales ya definidos.
- [ ] El diseño es responsive (móvil, tablet, desktop)
- [ ] Se respetan los colores institucionales
- [ ] Las imágenes tienen atributo `alt`
- [ ] El código está comentado cuando es necesario
- [ ] Los nombres de variables/funciones son descriptivos
- [ ] Has probado en al menos 2 navegadores diferentes
