# Política de Seguridad (Interna)

Este repositorio es de uso interno FEUNA/DTIC.

## Reporte de incidentes

No publicar vulnerabilidades en issues públicos.

Reportar por canal interno FEUNA/DTIC incluyendo:

- Qué problema se detectó.
- Cómo reproducirlo.
- Impacto esperado.
- Evidencia básica (capturas o logs).

## Respuesta esperada

- Confirmación de recibido: dentro de 48 horas hábiles.
- Priorización y plan de solución: lo antes posible según severidad.

## Reglas mínimas de seguridad

- No subir tokens, contraseñas ni claves API al repositorio.
- Revisar variables y secretos antes de hacer commit.
- Mantener dependencias y workflows actualizados.
- Aplicar principio de mínimo privilegio en accesos.
